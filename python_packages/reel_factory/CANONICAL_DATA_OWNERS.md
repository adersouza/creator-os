# Canonical Data Owners

Purpose: define the system of record for major Creator OS objects so future work does not create competing sources of truth.

Rule of thumb: exports, sidecars, mirrors, and UI metadata are not automatically canonical. The canonical owner is the place that creates the object and is allowed to make primary updates.

## Ownership Rules

- Pipeline Contracts owns schema shape, not object state.
- ContentForge owns audit reports and variant outputs, not campaign/post truth.
- Reel Factory owns generation artifacts until they are imported or exported into campaign/app systems.
- Campaign Factory owns campaign control decisions and draft export packages.
- ThreadsDashboard owns real app posts, account data, approvals, schedules, publishing status, and platform metrics.
- The Reel Factory posting ledger is a local Campaign Factory-compatible pilot planning source. For the 5-account pilot and beyond, promote it with Campaign Factory `promote-reel-ledger`; after promotion, Campaign Factory `distribution_plans` are the canonical campaign schedule/account state and ThreadsDashboard remains the owner of actual app post schedule rows.
